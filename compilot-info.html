Compilot

Compilot compiles our own programming language by making a lexical analyzer (compilot.lex), a parser (compilot.y), and bash script to build the compiler (compilot.sh). The lexical analyzer is written in Flex and tokenizes input source code into meaningful tokens, as well as handles errors from unexpected input characters. The parser is written using Bison and defines the grammar rules of the compilot programming language. Using the tokens generated by the lexer, it builds an abstract syntax tree for the program.

Skills and methods:

  • C++: The programming language used to construct the lexical analyzer and parser.
  • Regular expressions: Used to match and tokenize different types of tokens in the lexer.
  • Abstract Syntax Tree: Familiarity with constructing and manipulating ASTs in the parser to represent the syntactic structure of the source code.
  • Semantic Analysis: Performing semantic analysis in the parser to enforce language-specific rules and constraints.
